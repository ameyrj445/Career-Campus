# 1. INTRODUCTION

## 1.1 Background of the Project

In today's competitive job market, students and professionals need efficient ways to discover relevant job and internship opportunities. Traditional job boards often lack intuitive filtering capabilities and user-friendly interfaces. This project addresses these limitations by creating a modern, database-driven job and internship portal that emphasizes ease of use, comprehensive filtering, and an engaging user interface.

The portal serves as a centralized platform where users can browse through various job and internship listings, filter them based on multiple criteria, and access detailed information about each opportunity. The system demonstrates practical implementation of database management concepts, including relational database design, complex SQL queries, and integration with a modern web frontend.

## 1.2 Objectives

The primary objectives of this project are:

1. To design and implement a relational database system that efficiently stores and manages job and internship listings
2. To develop a responsive web interface that allows users to browse and filter listings based on various criteria
3. To implement advanced filtering capabilities including location, salary/stipend, company, and skills/tags
4. To create an interactive card-based UI that displays comprehensive information about each listing
5. To demonstrate practical application of database concepts including schema design, normalization, and complex SQL queries
6. To build a scalable system architecture with clear separation between frontend, backend, and database layers

## 1.3 Scope of the Project

The scope of this project encompasses:

1. Design and implementation of a relational database to store job and internship listings
2. Development of a backend API to handle data retrieval and filtering
3. Creation of a responsive frontend interface for browsing and filtering listings
4. Implementation of interactive job/internship cards with flip animation to show detailed information
5. Development of a comprehensive filtering system with multiple filter options
6. Implementation of search functionality across both job and internship listings

The project does not include user authentication, application submission, or administrative interfaces for managing listings, as these would be considered extensions for future development.

# 2. SYSTEM ANALYSIS

## 2.1 Problem Statement

Students and job seekers often struggle to find relevant opportunities due to:

1. Scattered information across multiple platforms
2. Poor filtering capabilities on existing job portals
3. Lack of standardized presentation of job/internship details
4. Difficulty in comparing opportunities based on specific criteria
5. Unintuitive user interfaces that hinder efficient browsing

This project aims to address these challenges by creating a centralized platform with advanced filtering capabilities and a user-friendly interface that makes it easy to discover and compare job and internship opportunities.

## 2.2 Functional Requirements

The system must provide the following functionalities:

1. **Listing Display**:
   - Display job and internship listings in an organized, card-based format
   - Show essential information on the front of cards (title, company, location, compensation)
   - Provide detailed information on the back of cards (description, requirements, application link)
   - Support flipping animation to toggle between front and back views

2. **Filtering Capabilities**:
   - Filter by location (city)
   - Filter by salary/stipend range
   - Filter by company name
   - Filter by job category
   - Filter by required skills/tags
   - Filter by experience level (for jobs)
   - Filter by duration (for internships)
   - Filter by education requirements (for jobs)
   - Filter by remote work option

3. **Search Functionality**:
   - Search across both job and internship listings
   - Search by keywords in title, company, category, or tags

4. **Navigation**:
   - Switch between jobs and internships views
   - Clear all applied filters
   - Apply multiple filters simultaneously

## 2.3 Non-Functional Requirements

1. **Performance**:
   - Fast loading of listings (< 2 seconds)
   - Responsive filtering without page reloads
   - Efficient database queries optimized for filtering operations

2. **Usability**:
   - Intuitive and clean user interface
   - Mobile-responsive design
   - Consistent visual styling
   - Clear presentation of filter options

3. **Reliability**:
   - Proper error handling for database operations
   - Graceful degradation when filters return no results

4. **Scalability**:
   - Database design that accommodates growth in listings
   - Modular code structure for easy maintenance and extension

5. **Security**:
   - Protection against SQL injection through prepared statements
   - Input validation for all filter parameters

# 3. SYSTEM DESIGN

## 3.1 ER Diagram

The Entity-Relationship diagram for the Job and Internship Portal consists of two main entities:

1. **Job_Listings**: Represents all job opportunities
2. **Internship_Listings**: Represents all internship opportunities

These entities have similar attributes but are kept separate due to their distinct characteristics and filtering requirements.

[Note: An actual ER diagram would be included in the final report]

## 3.2 Schema Design

The database schema follows a simple design with two primary tables:

1. **job_listings**: Stores all job-related information
2. **internship_listings**: Stores all internship-related information

This design was chosen over a normalized approach with separate tables for companies, locations, etc., to optimize for read performance and simplify queries, as the application is primarily read-oriented.

## 3.3 Table Structure

### job_listings Table

```
CREATE TABLE job_listings (
    id INT PRIMARY KEY AUTO_INCREMENT,
    title_and_role VARCHAR(255) NOT NULL,
    company_name VARCHAR(255) NOT NULL,
    location VARCHAR(255),
    category VARCHAR(100),
    salary_LPA DECIMAL(10,2),
    experience VARCHAR(50),
    education_level VARCHAR(100),
    date_posted DATE,
    tags TEXT,
    description TEXT,
    requirements TEXT
);
```

### internship_listings Table

```
CREATE TABLE internship_listings (
    ID INT PRIMARY KEY AUTO_INCREMENT,
    TitleAndRole VARCHAR(255) NOT NULL,
    CompanyName VARCHAR(255) NOT NULL,
    Location VARCHAR(255),
    Category VARCHAR(100),
    Stipend DECIMAL(10,2),
    Duration INT,
    Remote BOOLEAN,
    DatePosted DATE,
    Tags TEXT,
    Description TEXT,
    Requirements TEXT
);
```

## 3.4 Relationship Definitions

While the current implementation uses two separate tables without explicit relationships, the design could be extended to include:

1. A companies table with a one-to-many relationship to both job and internship listings
2. A locations table with a one-to-many relationship to both job and internship listings
3. A categories table with a one-to-many relationship to both job and internship listings
4. A tags table with a many-to-many relationship to both job and internship listings

These relationships would improve data integrity and reduce redundancy but were not implemented in the current version to prioritize query performance and simplicity.

## 4.2 System Architecture

The application follows a three-tier architecture:

1. **Presentation Layer (Frontend)**:
   - React components for UI rendering
   - Context API for state management
   - Axios for API communication

2. **Application Layer (Backend)**:
   - Express.js server handling HTTP requests
   - API endpoints for data retrieval and filtering
   - Business logic for query construction

3. **Data Layer (Database)**:
   - MySQL database storing job and internship data
   - SQL queries for data retrieval and filtering

The architecture ensures separation of concerns, with each layer responsible for specific functionality:
- Frontend handles user interaction and display
- Backend processes requests and constructs database queries
- Database stores and retrieves data efficiently

## 4.3 Database Implementation

The database implementation includes:

The project uses various SQL queries for data retrieval and filtering. Key queries include:

### Base Job Query with Filters
```sql
SELECT * FROM job_listings 
WHERE 1=1 
AND (location = ? OR location = ? OR ...)
AND salary_LPA >= ?
AND (category LIKE ? OR category LIKE ? OR ...)
AND (company_name LIKE ? OR company_name LIKE ? OR ...)
ORDER BY date_posted DESC
```

### Base Internship Query with Filters
```sql
SELECT * FROM internship_listings 
WHERE 1=1 
AND (Location = ? OR Location = ? OR ...)
AND Stipend >= ?
AND (Category LIKE ? OR Category LIKE ? OR ...)
AND (CompanyName LIKE ? OR CompanyName LIKE ? OR ...)
ORDER BY DatePosted DESC
```

### Search Query Across Both Tables
```sql

    TitleAndRole LIKE ? OR

SELECT 
    'job' as type,
    id,

    category,
```

## 8.2 API Endpoints

The backend provides the following API endpoints:

### Job Endpoints
- `GET /api/jobs`: Get all jobs with optional filters
- `GET /api/jobs/:id`: Get a specific job by ID
- `GET /api/job-filter-options`: Get all filter options for jobs

### Internship Endpoints
- `GET /api/internships`: Get all internships with optional filters
- `GET /api/internships/:id`: Get a specific internship by ID
- `GET /api/internship-filter-options`: Get all filter options for internships

### Utility Endpoints
- `GET /api/categories`: Get all unique categories
- `GET /api/locations`: Get all unique locations
- `GET /api/companies`: Get all unique companies
- `GET /api/tags`: Get all unique tags
- `GET /api/search`: Search across both jobs and internships
