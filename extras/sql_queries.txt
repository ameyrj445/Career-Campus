## Database Schema

The project uses two main tables:

### job_listings Table
```sql
CREATE TABLE job_listings (
    id INT PRIMARY KEY AUTO_INCREMENT,
    title_and_role VARCHAR(255) NOT NULL,
    company_name VARCHAR(255) NOT NULL,
    location VARCHAR(255),
    category VARCHAR(100),
    salary_LPA DECIMAL(10,2),
    experience VARCHAR(50),
    education_level VARCHAR(100),
    date_posted DATE,
    tags TEXT,
    description TEXT,
    requirements TEXT
);
```

### internship_listings Table
```sql
CREATE TABLE internship_listings (
    ID INT PRIMARY KEY AUTO_INCREMENT,
    TitleAndRole VARCHAR(255) NOT NULL,
    CompanyName VARCHAR(255) NOT NULL,
    Location VARCHAR(255),
    Category VARCHAR(100),
    Stipend DECIMAL(10,2),
    Duration INT,
    Remote BOOLEAN,
    DatePosted DATE,
    Tags TEXT,
    Description TEXT,
    Requirements TEXT
);
```

## Job Listing Queries

### Get All Jobs (with Filters)
This query retrieves job listings with optional filters. The base query is enhanced with conditions based on user-selected filters.

**Base Query:**
```sql
SELECT * FROM job_listings WHERE 1=1
```

**With Location Filter:**
```sql
SELECT * FROM job_listings 
WHERE 1=1 
AND (location = ? OR location = ? OR ...)
ORDER BY date_posted DESC
```

**With Salary Filter:**
```sql
SELECT * FROM job_listings 
WHERE 1=1 
AND salary_LPA >= ?
ORDER BY date_posted DESC
```

**With Category Filter:**
```sql
SELECT * FROM job_listings 
WHERE 1=1 
AND (category LIKE ? OR category LIKE ? OR ...)
ORDER BY date_posted DESC
```

**With Company Filter:**
```sql
SELECT * FROM job_listings 
WHERE 1=1 
AND (company_name LIKE ? OR company_name LIKE ? OR ...)
ORDER BY date_posted DESC
```

**With Experience Filter:**
```sql
SELECT * FROM job_listings 
WHERE 1=1 
AND experience LIKE ?
ORDER BY date_posted DESC
```

**With Remote Filter:**
```sql
SELECT * FROM job_listings 
WHERE 1=1 
AND remote = ?
ORDER BY date_posted DESC
```

**With Education Filter:**
```sql
SELECT * FROM job_listings 
WHERE 1=1 
AND (education_level LIKE ? OR education_level LIKE ? OR ...)
ORDER BY date_posted DESC
```

**With Tags Filter:**
```sql
SELECT * FROM job_listings 
WHERE 1=1 
AND (
    (tags LIKE ? OR tags LIKE ? OR ...) -- For tag at beginning
    OR (tags LIKE ? OR tags LIKE ? OR ...) -- For tag in middle
    OR (tags LIKE ? OR tags LIKE ? OR ...) -- For tag at end
    OR (tags = ? OR tags = ? OR ...) -- For exact match
)
ORDER BY date_posted DESC
```

**Complete Example with Multiple Filters:**
```sql
SELECT * FROM job_listings 
WHERE 1=1 
AND (location = 'Bangalore, India' OR location = 'Mumbai, India')
AND salary_LPA >= 20
AND (category LIKE '%Technology%' OR category LIKE '%IT%')
AND (company_name LIKE '%Google%' OR company_name LIKE '%Microsoft%')
AND experience LIKE '%3-5 years%'
AND (
    tags LIKE 'React,%' OR 
    tags LIKE '%, React,%' OR 
    tags LIKE '%, React' OR 
    tags = 'React'
)
ORDER BY date_posted DESC
```

## Search Queries

### Search Across Both Jobs and Internships
This query searches for a term across both job and internship listings, combining the results with a UNION.

```sql
SELECT 
    'internship' as type,
    ID as id,
    TitleAndRole as title,
    CompanyName as company,
    Location as location,
    Category as category,
    Stipend as compensation,
    DatePosted as posted_date,
    Tags as tags
FROM internship_listings
WHERE 
    TitleAndRole LIKE ? OR
    CompanyName LIKE ? OR
    Category LIKE ? OR
    Tags LIKE ?
UNION ALL
SELECT 
    'job' as type,
    id,
    title_and_role as title,
    company_name as company,
    location,
    category,
    salary_LPA as compensation,
    date_posted as posted_date,
    tags
FROM job_listings
WHERE 
    title_and_role LIKE ? OR
    company_name LIKE ? OR
    category LIKE ? OR
    tags LIKE ?
ORDER BY posted_date DESC
```

## Query Implementation Details

### Dynamic Query Building
The application builds SQL queries dynamically based on user-selected filters. This is done by:

1. Starting with a base query: `SELECT * FROM job_listings WHERE 1=1`
2. Adding conditions based on filters: `AND (condition1) AND (condition2) ...`
3. Adding parameters to a values array for prepared statements
4. Executing the final query with the values array

### Prepared Statements
All queries use prepared statements to prevent SQL injection:

```javascript
const [rows] = await pool.execute(query, values);
```

### Location Handling
Location values are standardized to the format "city, India" for consistent filtering:

```javascript
// If location doesn't already end with India, add it for the search
if (!/,\s*India$/i.test(cleanLoc)) {
    values.push(`${cleanLoc}, India`);
} else {
    values.push(cleanLoc);
}
```

### Tag Filtering
Tag filtering is complex because tags are stored as comma-separated strings. The query handles different positions of tags:

```javascript
// Match at the beginning
tagConditions.push('tags LIKE ?');
tagValues.push(`${tag},%`);

// Match in the middle
tagConditions.push('tags LIKE ?');
tagValues.push(`%, ${tag},%`);

// Match at the end
tagConditions.push('tags LIKE ?');
tagValues.push(`%, ${tag}`);

// Match if it's the only tag
tagConditions.push('tags = ?');
tagValues.push(tag);
```

## Performance Considerations

1. **Indexes**: The database should have indexes on frequently filtered columns:
   ```sql
   CREATE INDEX idx_job_location ON job_listings(location);
   CREATE INDEX idx_job_category ON job_listings(category);
   CREATE INDEX idx_job_company ON job_listings(company_name);
   CREATE INDEX idx_job_date ON job_listings(date_posted);
   
   CREATE INDEX idx_internship_location ON internship_listings(Location);
   CREATE INDEX idx_internship_category ON internship_listings(Category);
   CREATE INDEX idx_internship_company ON internship_listings(CompanyName);
   CREATE INDEX idx_internship_date ON internship_listings(DatePosted);
   ```

2. **Text Searching**: For better performance with tag filtering, consider using a full-text search index:
   ```sql
   ALTER TABLE job_listings ADD FULLTEXT(tags);
   ALTER TABLE internship_listings ADD FULLTEXT(Tags);
   ```

3. **Query Optimization**: The application uses `ORDER BY date_posted DESC` to show the most recent listings first, which benefits from an index on the date column.

## Conclusion

The SQL queries in this project are designed to provide flexible filtering and searching capabilities across job and internship listings. The dynamic query building approach allows for complex combinations of filters while maintaining security through prepared statements.

The standardization of location data and special handling for tag filtering ensures consistent results across the application. Performance considerations like indexing are important for maintaining good response times as the database grows.
