Job/Internship Portal - Project Architecture

This diagram illustrates the architecture and data flow of the Job/Internship Portal application. The application follows a three-tier architecture with a React frontend, Node.js/Express backend, and MySQL database.

System Architecture

Frontend (React)

User Interface, Filtering, Card Display

Backend (Node.js/Express)

API Endpoints, Business Logic, Data Processing

Database (MySQL)

job_listings, internship_listings tables

Component Structure

App.js

Main Application Component

JobContext.js

State Management, API Calls

JobsPage.js

Jobs Listing Page

InternshipsPage.js

Internships Listing Page

JobFilters.js

Filtering UI for Jobs

JobCard.js

Job Card Component

InternshipFilters.js

Filtering UI for Internships

Data Flow

1

User Interaction

User visits the Jobs or Internships page and may apply filters

2

Frontend Request

React components make API calls to the backend with filter parameters

3

Backend Processing

Express routes handle requests and query the database with filter conditions

4

Database Query

SQL queries filter data based on user selections (location, tags, salary, etc.)

5

Response Processing

Backend formats database results and sends JSON response

6

UI Rendering

React components render job/internship cards with the filtered data

Key Features

  • Flippable job/internship cards with front and back views
  • Advanced filtering system with multiple filter options
  • Location filtering with standardized "city, India" format
  • Tag filtering with horizontal scrolling in cards
  • Responsive design for mobile and desktop
  • Sharp-edged scrollbars for consistent UI

Database Schema

job_listings Table

id INT PRIMARY KEY AUTO_INCREMENT,
title_and_role VARCHAR(255) NOT NULL,
company_name VARCHAR(255) NOT NULL,
location VARCHAR(255),
category VARCHAR(100),
salary_LPA DECIMAL(10,2),
experience VARCHAR(50),
education_level VARCHAR(100),
date_posted DATE,
tags TEXT,
description TEXT,
requirements TEXT

internship_listings Table

ID INT PRIMARY KEY AUTO_INCREMENT,
TitleAndRole VARCHAR(255) NOT NULL,
CompanyName VARCHAR(255) NOT NULL,
Location VARCHAR(255),
Category VARCHAR(100),
Stipend DECIMAL(10,2),
Duration INT,
Remote BOOLEAN,
DatePosted DATE,
Tags TEXT,
Description TEXT,
Requirements TEXT
